Отчет по реализации проекта Multi-Agent Interview Coach
Обзор проекта
Данный проект представляет собой мультиагентную систему для проведения технических интервью, разработанную в соответствии с техническим заданием. Система реализует архитектуру из двух специализированных агентов, взаимодействующих для проведения адаптивного технического собеседования.

Соответствие требованиям технического задания
Система полностью соответствует всем обязательным требованиям:

Ролевая специализация: Реализованы два отдельных агента:

InterviewerAgent - ведет диалог и задает вопросы

ObserverAgent - анализирует ответы и дает рекомендации

Скрытая рефлексия: Реализован механизм внутреннего диалога между агентами перед генерацией ответа пользователю. Все "мысли" агентов записываются в лог.

Сохранение контекста: Система запоминает историю диалога, предотвращает повторные вопросы и учитывает предыдущие ответы кандидата.

Адаптивность: Сложность вопросов динамически меняется (junior/middle/senior) в зависимости от качества ответов кандидата.

Устойчивость к вбросам: Система распознает галлюцинации и попытки ухода от темы, корректно возвращая беседу в нужное русло.

Архитектура системы
Проект реализует модульную архитектуру:

Основные компоненты
InterviewCoordinator - центральный компонент, управляющий взаимодействием агентов, состоянием интервью и адаптацией сложности.

InterviewerAgent - агент, непосредственно общающийся с кандидатом, генерирующий вопросы на основе контекста и рекомендаций Observer.

ObserverAgent - агент-аналитик, оценивающий ответы кандидата по качеству, наличию ошибок и релевантности.

InterviewMemory - система памяти, хранящая историю диалога, контекст интервью и статистику по темам.

MistralClient - единый клиент для работы с Mistral LLM API.

InterviewJSONLogger - логгер, сохраняющий сессии в формате, соответствующем требованиям задания.

Взаимодействие компонентов
Координатор управляет потоком интервью: получает ответ кандидата, передает его Observer для анализа, организует внутренний диалог между агентами, а затем направляет рекомендации Interviewer для формирования следующего вопроса. Весь процесс логируется с сохранением как видимых сообщений, так и внутренних "мыслей" агентов.

Ключевые особенности реализации
Полное соответствие формату логов
Система генерирует логи в точном соответствии с требованиями задания, включая поля team_name, turns с внутренними мыслями агентов и структурированный final_feedback.

Динамическая адаптация
На основе оценки качества ответа (1-10 баллов) система автоматически регулирует сложность последующих вопросов, реализуя индивидуальный подход к каждому кандидату.

Обработка сложных сценариев
Система корректно обрабатывает:

Фактические ошибки и галлюцинации в ответах

Попытки сменить тему обсуждения

Встречные вопросы от кандидата

Команды завершения интервью

Генерация структурированного фидбэка
По завершении интервью система формирует развернутый отчет, включающий:

Вердикт с уровнем кандидата и рекомендацией по найму

Анализ hard skills с подтвержденными навыками и пробелами

Оценку soft skills (ясность изложения, честность, вовлеченность)

Персональный roadmap для развития навыков


Инструкция по запуску
Установка зависимостей
pip install -r requirements.txt

Создать файл .env с указанием API ключа:
MISTRAL_API_KEY=ваш_ключ
Запуск системы
python src/run_interview.py
